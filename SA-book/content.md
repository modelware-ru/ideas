# Теория и практика системного анализа при разработке программных систем

## Содержание

- Место системного анализа в процессе разработки программных систем
    - Ролевая модель
    - Ответственности и артефакты
- Роли, с которыми взаимодействует системный аналитик
    - Бизнес-аналитик
    - Дизайнер
    - Разработчик
    - Тестировщик
    - Системный администратор
    - Пользователь системы
- Входные артефакты. Данные для системного анализа
    - Бизнес-требования
        - User Story
        - Job Story
        - User Story Mapping
        - ...
- Выходные артефакты. Результат системного анализа
    - Модель предметной области
    - Словарь терминов, глоссарий
    - Ролевая модель
    - Модель использования
        - Сценарии использования (use case)
    - Схема пользовательского интерфейса
        - структура (компоненты пользовательского интерфейса, mockups)
        - поведение (взаимодействие между экранами)
    - Схема базы данных
        - концептуальный
        - логический
        - физический уровень
    - Компонентная модель
    - Модель развертывания
- Программная система. Модель предметной области
- Практика разработки программной система
    - Моделирование использования
    - Моделирование структуры (структурная декомпозиция)
    - Моделирование поведения (функциональная декомпозиция)
- Сквозной пример. Сервис отправки почтовых сообщений
